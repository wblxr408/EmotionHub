#### 5. 实现JWT认证

**学习目标**：掌握无状态认证机制和Spring Security集成

**关键概念**：
- JWT（JSON Web Token）结构
- Spring Security过滤器链
- 认证（Authentication）vs 授权（Authorization）

**学习资源**：
- [JWT官方网站](https://jwt.io/)
- [Spring Security官方文档](https://spring.io/projects/spring-security)
- [JWT Java库 - jjwt](https://github.com/jwtk/jjwt)

**实践步骤**：
1. 添加JWT依赖
   ```xml
   <dependency>
       <groupId>io.jsonwebtoken</groupId>
       <artifactId>jjwt-api</artifactId>
   </dependency>
   ```
2. 创建JWT工具类
   ```java
   public class JwtUtil {
       public String generateToken(String username) {
           return Jwts.builder()
               .setSubject(username)
               .setExpiration(new Date(System.currentTimeMillis() + expiration))
               .signWith(key)
               .compact();
       }

       public String parseToken(String token) {
           return Jwts.parserBuilder()
               .setSigningKey(key)
               .build()
               .parseClaimsJws(token)
               .getBody()
               .getSubject();
       }
   }
   ```
3. 创建JWT过滤器
   ```java
   public class JwtAuthenticationFilter extends OncePerRequestFilter {
       @Override
       protected void doFilterInternal(HttpServletRequest request,
                                       HttpServletResponse response,
                                       FilterChain chain) {
           String token = request.getHeader("Authorization");
           if (token != null && token.startsWith("Bearer ")) {
               String username = jwtUtil.parseToken(token.substring(7));
               // 设置认证信息到SecurityContext
           }
           chain.doFilter(request, response);
       }
   }
   ```
4. 配置Spring Security
   ```java
   @Configuration
   @EnableWebSecurity
   public class SecurityConfig {
       @Bean
       public SecurityFilterChain filterChain(HttpSecurity http) {
           http.csrf().disable()
               .authorizeHttpRequests(auth -> auth
                   .requestMatchers("/user/login", "/user/register").permitAll()
                   .anyRequest().authenticated()
               )
               .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
           return http.build();
       }
   }
   ```

**验证方式**：
- 调用登录接口，获取JWT Token
- 使用Token访问受保护的接口
- 测试Token过期和无效Token的情况

#### 4. 配置路由和状态管理

**学习目标**：理解SPA路由机制和全局状态管理

**关键概念**：
- Vue Router路由守卫
- Pinia状态管理（Vuex的替代品）
- 路由懒加载

**学习资源**：
- [Vue Router官方文档](https://router.vuejs.org/)
- [Pinia官方文档](https://pinia.vuejs.org/)

**实践步骤**：

**路由配置**：
1. 安装依赖
   ```bash
   npm install vue-router@4
   ```
2. 创建`src/router/index.ts`
   ```ts
   import { createRouter, createWebHistory } from 'vue-router'

   const routes = [
     {
       path: '/',
       name: 'Home',
       component: () => import('@/views/Home.vue')
     },
     {
       path: '/login',
       name: 'Login',
       component: () => import('@/views/Login.vue')
     }
   ]

   const router = createRouter({
     history: createWebHistory(),
     routes
   })

   // 路由守卫
   router.beforeEach((to, from, next) => {
     const token = localStorage.getItem('token')
     if (!token && to.path !== '/login') {
       next('/login')
     } else {
       next()
     }
   })

   export default router
   ```
3. 在`main.ts`中注册
   ```ts
   import router from './router'
   app.use(router)
   ```

**状态管理**：
1. 安装依赖
   ```bash
   npm install pinia
   ```
2. 创建`src/stores/user.ts`
   ```ts
   import { defineStore } from 'pinia'

   export const useUserStore = defineStore('user', {
     state: () => ({
       token: '',
       userInfo: null
     }),
     actions: {
       setToken(token: string) {
         this.token = token
         localStorage.setItem('token', token)
       },
       logout() {
         this.token = ''
         this.userInfo = null
         localStorage.removeItem('token')
       }
     }
   })
   ```
3. 在组件中使用
   ```vue
   <script setup lang="ts">
   import { useUserStore } from '@/stores/user'
   const userStore = useUserStore()
   </script>
   ```

**验证方式**：
- 测试路由跳转是否正常
- 测试路由守卫是否生效
- 测试Pinia状态是否能正常读写

#### 5. 封装Axios请求

**学习目标**：掌握HTTP请求封装和拦截器使用

**关键概念**：
- Axios请求/响应拦截器
- 统一错误处理
- 请求取消和超时控制

**学习资源**：
- [Axios官方文档](https://axios-http.com/)
- [Axios拦截器](https://axios-http.com/docs/interceptors)

**实践步骤**：
1. 安装依赖
   ```bash
   npm install axios
   ```
2. 创建`src/utils/request.ts`
   ```ts
   import axios from 'axios'
   import { ElMessage } from 'element-plus'
   import { useUserStore } from '@/stores/user'

   const request = axios.create({
     baseURL: '/api',
     timeout: 10000
   })

   // 请求拦截器
   request.interceptors.request.use(
     config => {
       const userStore = useUserStore()
       if (userStore.token) {
         config.headers.Authorization = `Bearer ${userStore.token}`
       }
       return config
     },
     error => {
       return Promise.reject(error)
     }
   )

   // 响应拦截器
   request.interceptors.response.use(
     response => {
       const { code, message, data } = response.data
       if (code === 200) {
         return data
       } else {
         ElMessage.error(message || '请求失败')
         return Promise.reject(new Error(message))
       }
     },
     error => {
       if (error.response?.status === 401) {
         ElMessage.error('登录已过期，请重新登录')
         // 跳转到登录页
       } else {
         ElMessage.error(error.message || '网络错误')
       }
       return Promise.reject(error)
     }
   )

   export default request
   ```
3. 创建API接口文件`src/api/user.ts`
   ```ts
   import request from '@/utils/request'

   export interface LoginDTO {
     username: string
     password: string
   }

   export const userApi = {
     login(data: LoginDTO) {
       return request.post('/user/login', data)
     },
     getUserInfo() {
       return request.get('/user/info')
     }
   }
   ```

**验证方式**：
- 调用API接口，检查请求头是否携带Token
- 测试401错误是否能正确跳转登录页
- 测试网络错误是否有提示

## 常见问题

### Q1: Maven依赖下载失败怎么办？
**A**: 配置国内镜像源（阿里云Maven仓库）
```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### Q2: npm安装依赖很慢怎么办？
**A**: 使用国内镜像源
```bash
npm config set registry https://registry.npmmirror.com
```

### Q3: Spring Boot启动报错找不到主类？
**A**: 检查`emotion-hub-web`模块的`pom.xml`是否配置了`spring-boot-maven-plugin`

### Q4: 前端跨域问题怎么解决？
**A**: 在`vite.config.ts`配置代理，或者后端添加CORS配置

### Q5: JWT Token存储在哪里？
**A**: 推荐存储在`localStorage`，但要注意XSS攻击风险
