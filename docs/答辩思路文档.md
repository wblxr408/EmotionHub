# EmotionHub

### 基于图神经网络情感传染模型与实时 Feed 流优化的社交媒体深度分析与推荐平台

> *"在情绪的海洋里，我们为你找到属于你的那一束光。"*

---

**一句话介绍**：EmotionHub 是一个将"情感传染理论"融入社交推荐系统的平台，让算法不仅读懂你的兴趣，更读懂你此刻的心情。

## 一、开发背景

### 1.1 问题起源

在当下的社交媒体环境中，用户每天被动接收大量内容，但平台推荐逻辑几乎只围绕"用户感兴趣什么"展开——谁在意过你**此刻的心情**？

一个真实场景：当用户在经历情绪低落时，平台仍然推送引发焦虑的负面内容；当用户心情愉悦时，却因为"协同过滤"的茧房效应，持续困在单一情感维度的信息中。

更深层的问题是：**情感在社交网络中如何流动？哪些人更容易影响他人的情绪？相似的情感倾向是否能自动汇聚成社区？**

这些在现有社交平台中几乎无人回答。

### 1.2 我们想做什么

EmotionHub 是一个以**情感智能**为核心的社交媒体深度分析平台：

- **对普通用户**：提供"情感自适应"的智能 Feed 流，不再被算法困在情绪茧房里
- **对研究者**：提供情感传播链追踪、用户影响力量化、情感社区发现等分析工具
- **对平台运营者**：提供多维情感统计和可管控的内容生态

### 1.3 技术挑战

| 挑战               | 描述                                           |
| ------------------ | ---------------------------------------------- |
| 如何量化"情感"本身 | 将文本内容转化为可计算的情感分数与标签体系     |
| 如何建模情感传播   | 评论如何继承或反转帖子的情感？传播链如何追踪？ |
| 如何发现情感社区   | 无监督地从海量交互中自动发现情感相似的用户群体 |
| 如何融合情感与推荐 | 将情感维度嵌入推荐算法，而非简单叠加           |
| 如何保证实时性     | Feed 流低延迟 + 情感状态实时更新               |

## 四、主要创新点

> *"算法写下的，是情感的方程式；我们求解的，是人心的温度。"*

### 创新点一：用户情感状态自适应推荐引擎

**核心思想**：推荐系统不仅要懂"你兴趣什么"，更要懂"你此刻心情如何"。

我们设计了完整的**情感状态识别 → 内容情感匹配 → 推荐策略选择**闭环：

```
用户情感状态识别
│
├── 滑动窗口统计（最近 24h 帖子情感分数加权均值）
│
└── 状态分类机
      ├── LOW（情绪低落）
      │     → 策略：推送高正向情感内容（治愈系、正能量）
      │
      ├── ANXIOUS（焦虑）
      │     → 策略：推送中性平静内容（减少情感刺激）
      │
      ├── HAPPY（愉悦）
      │     → 策略：全量推送（情感多元化，防止茧房）
      │
      ├── CALM（平静）
      │     → 策略：推送温和积极内容（保持情绪稳定）
      │
      └── FLUCTUANT（波动）
            → 策略：推送情感中性内容（稳定情绪）
```

**互补疗愈推荐公式**：

```
content.emotionScore > 0.6  &&  tag in [POSITIVE_ENERGY, HEALING]  →  推荐给 LOW 状态用户
|emotionScore| < 0.2  &&  tag == NEUTRAL_CALM                    →  推荐给 ANXIOUS 用户
```

最终推荐评分：`finalScore = 0.5 × 热度分 + 0.3 × 情感匹配分 + 0.2 × 时效分`

---

### 创新点二：Louvain 算法驱动的情感社区发现

**核心思想**：具有相似情感特征的人会自动聚集成社区——我们用算法把这件事"白盒化"。

**算法流程**：

```
Step 1: 情感特征向量构建
        每个用户 → 4维向量 [avgScore, positiveRatio, neutralRatio, negativeRatio]

Step 2: 余弦相似度计算
        sim(A, B) = dot(vA, vB) / (|vA| × |vB|)
        归一化至 [0, 1]

Step 3: 共鸣边构建
        sim ≥ 0.3 → 建立无向边，权重 = sim分值

Step 4: Louvain 社区发现（贪心变体）
        迭代地将每个节点移入使"社区内总权重增加最大"的社区
        迭代上限 30 次，防止死循环

Step 5: 社区分类
        avgScore > 0.2 → 乐观派（OPTIMISTIC）
        avgScore < -0.2 → 悲观派（PESSIMISTIC）
        其余 → 理性派（RATIONAL）
```

**实际意义**：平台可以清晰看到"悲观派社区"的用户构成，从而在内容推荐时进行精准的情感干预。

---

### 创新点三：情感传播链建模与转折点识别

**核心思想**：每一层评论对原帖情感做了什么？是被"放大"还是被"衰减"？在哪里发生了情感反转？

**核心指标**：

| 指标                   | 计算公式                     | 含义                     |
| ---------------------- | ---------------------------- | ------------------------ |
| **情感一致性**   | `1 -                         | postScore - commentScore |
| **情感放大系数** | `commentScore / postScore` | 评论情感是否被放大       |
| **情感转折点**   | 正面↔负面符号变化           | 在哪条评论发生了反转     |

**传播类型判定**：

```
高一致性 + 高放大 → AMPLIFIED（放大传播）：情感被评论进一步强化
高一致性 + 低放大 → CONSISTENT（一致传播）：情感稳定传递
低一致性 + 多转折 → CONTROVERSIAL（争议传播）：引发情感分化
低放大系数       → DAMPENED（衰减传播）：情感逐渐消散
```

### 创新点四：协同过滤 + 情感感知双塔推荐

**核心思想**：推荐既要看"你跟谁口味相似"，也要看"你跟谁情感相投"。

**技术实现**：

```
候选集生成层
│
├── Redis 缓存层（每日凌晨 2:30 自动重建）
│     ├── 用户个性化推荐列表（ZSet，Top 200）
│     └── 热点帖子排行
│
├── ALS 矩阵分解（10维隐向量，8次迭代）
│     └── 用户-帖子交互矩阵分解
│
└── 推荐评分融合
      ├── 兴趣相似度（余弦，权重 60%）
      │     基于点赞/评论/发帖行为矩阵
      └── 情感向量相似度（余弦，权重 40%）
            基于 [avgScore, posRatio, neuRatio, negRatio]

多样性保障
├── 同一作者最多占 25%
├── 同一情感标签最多占 60%
└── 剩余按 finalScore 降序补位
```

---

### 创新点五：多 LLM 集成的情感分析管道

**核心思想**：用 AI 大模型替代规则词典，提升情感分析的语义理解能力。

```
情感分析请求
      │
      ├── 模式1：关键词分析（默认，毫秒级响应）
      │     情感词典 + 否定词处理
      │
      └── 模式2：通义千问 LLM（配置 API Key 后启用）
            ├── Prompt 工程：角色设定 + 情感分类 + 分数输出
            ├── 负载均衡：轮询/随机/加权三种策略
            ├── 熔断降级：LLM 失败 → 自动回退关键词分析
            └── 结果缓存：24h TTL，同一内容不重复计费

支持扩展：
├── 文心一言（百度）
└── 智谱 GLM-4（国内）
```

## 六、项目亮点

> *"流畅如流水，规范如乐章，新颖如晨曦。"*

### 6.1 运行流畅

- **Redis 三级缓存**：热点帖子缓存（1小时）、用户推荐列表（ZSet）、情感分析结果（24h TTL），API 响应时间 < 100ms
- **异步任务池**：情感分析、推荐重算等耗时任务异步执行，用户无感知等待
- **定时任务自动化**：每日凌晨 2:30 自动重建协同过滤矩阵，每日 1:00 汇总情感统计，每周日凌晨 2:00 重算情感共鸣网络
- **连接池精细调优**：Druid（最大 20 连接）、Lettuce（最大 20 连接），高并发下稳定运行

### 6.2 整体协调

- **前后端完全分离**：RESTful API 解耦，前后端独立迭代，通过 Knife4j 在线调试
- **模块化 Maven 多模块**：common / model / dao / service / web 五层，职责边界清晰
- **Docker Compose 一键编排**：MySQL + Redis + Backend + Frontend 四个容器，接近生产级部署
- **统一返回格式**：`Result<T>` 封装所有接口响应，前端统一处理

### 6.3 开发规范

- **Flyway 数据库版本管理**：所有数据库变更通过 migration 脚本管理，团队协作无迁移冲突
- **统一异常处理**：`GlobalExceptionHandler` 全局捕获，返回标准化错误码（ErrorCode 枚举）
- **日志结构化分级**：DEBUG / INFO / WARN / ERROR 分级输出，生产可追踪
- **前端代码规范**：ESLint + Prettier 统一风格，TypeScript 类型守卫减少运行时错误
- **API 文档自动化**：Knife4j 自动生成 Swagger 文档，每次代码提交后可立即调试

### 6.4 创意新颖

| 维度               | 创新点                                                               |
| ------------------ | -------------------------------------------------------------------- |
| **理论创新** | 首次将"情感传染"理论应用于 Feed 流推荐，提出"情感状态自适应"推荐范式 |
| **算法创新** | 基于余弦相似度的情感共鸣网络 + Louvain 无监督社区发现，无需人工标注  |
| **工程创新** | 多 LLM 提供商统一抽象、熔断降级、24h 结果缓存，避免重复计费          |
| **交互创新** | ECharts 情感趋势折线图、情感社区力导向图、传播时间线图三屏联动       |
