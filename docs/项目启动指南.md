## 🗄️ 数据库初始化

### 1. 启动MySQL

**Windows:**
```bash
# 启动MySQL服务
net start MySQL80
```

**Mac/Linux:**
```bash
# 启动MySQL服务
sudo systemctl start mysql
# 或
brew services start mysql
```

### 2. 创建数据库并导入数据

```bash
# 登录MySQL
mysql -u root -p

# 执行初始化脚本
source database/init.sql

# 验证数据库创建成功
SHOW DATABASES;
# 应该能看到 emotion_hub 数据库

USE emotion_hub;
SHOW TABLES;
# 应该能看到7张表
```

### 3. 启动Redis

**Windows:**
```bash
redis-server
```

**Mac/Linux:**
```bash
redis-server
# 或后台启动
redis-server --daemonize yes
```

验证Redis启动：
```bash
redis-cli ping
# 应返回: PONG
```

## 🚀 后端启动

### 1. 修改配置文件

编辑 `backend/emotion-hub-web/src/main/resources/application-dev.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/emotion_hub?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的MySQL密码  # 修改这里

  data:
    redis:
      host: localhost
      port: 6379
      password:  # 如果Redis设置了密码，在这里填写
```

### 2. 编译项目

```bash
# 进入backend目录
cd backend

# 清理并编译
mvn clean install

# 如果编译成功，会显示 BUILD SUCCESS
```

### 3. 启动后端

**方式1: 使用Maven（推荐开发时使用）**
```bash
cd emotion-hub-web
mvn spring-boot:run
```

**方式2: 运行jar包**
```bash
java -jar emotion-hub-web/target/emotion-hub-web-1.0.0.jar
```

### 4. 验证后端启动

看到以下输出说明启动成功：
```
========================================
EmotionHub 启动成功！
API文档地址: http://localhost:8080/doc.html
========================================
```

测试接口：
```bash
curl http://localhost:8080/api/test/hello
```

应返回：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "Hello, EmotionHub!",
  "timestamp": 1707800000000
}
```

## 🎨 前端启动

### 1. 安装依赖

```bash
# 进入frontend目录
cd frontend

# 安装依赖（首次运行需要）
npm install

# 如果npm速度慢，可以使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### 2. 修改配置（可选）

编辑 `frontend/.env.development`

```env
# 后端API地址
VITE_API_BASE_URL=http://localhost:8080
```

### 3. 启动前端

```bash
npm run dev
```

看到以下输出说明启动成功：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

### 4. 访问前端

打开浏览器访问: http://localhost:3000

## ❓ 常见问题

### 1. 后端启动失败

**问题：端口8080被占用**
```
Port 8080 was already in use
```

解决方案：
- 修改 `application-dev.yml` 中的端口号
```yaml
server:
  port: 8081  # 改成其他端口
```

**问题：数据库连接失败**
```
Communications link failure
```

解决方案：
- 检查MySQL是否启动
- 检查用户名密码是否正确
- 检查数据库名称是否正确

**问题：Redis连接失败**
```
Unable to connect to Redis
```

解决方案：
- 检查Redis是否启动: `redis-cli ping`
- 如果Redis设置了密码，在配置文件中填写密码

### 2. 前端启动失败

**问题：依赖安装失败**
```
npm ERR! code ECONNREFUSED
```

解决方案：
```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```

**问题：端口3000被占用**

解决方案：
- 修改 `vite.config.ts`
```typescript
export default defineConfig({
  server: {
    port: 3001  // 改成其他端口
  }
})
```

### 3. 前后端连接失败

**问题：前端调用后端接口失败**

解决方案：
1. 确认后端已启动: http://localhost:8080/api/test/hello
2. 检查前端配置的API地址是否正确
3. 检查浏览器控制台是否有跨域错误
