# EmotionHub 项目启动指南

## 环境准备

### 必需软件

| 软件    | 版本要求 | 下载地址                                            |
| ------- | -------- | --------------------------------------------------- |
| JDK     | 17+      | https://www.oracle.com/java/technologies/downloads/ |
| Maven   | 3.6+     | https://maven.apache.org/download.cgi               |
| Node.js | 18+      | https://nodejs.org/                                 |
| MySQL   | 8.0+     | https://dev.mysql.com/downloads/mysql/              |
| Redis   | 7.0+     | https://redis.io/download                           |

### 验证安装

打开命令行，执行以下命令验证：

```bash
java -version
# 应显示: java version "17.x.x"

mvn -version
# 应显示: Apache Maven 3.x.x

node -v
# 应显示: v18.x.x 或更高

npm -v
# 应显示: 9.x.x 或更高

mysql --version
# 应显示: mysql Ver 8.0.x

redis-server --version
# 应显示: Redis server v=7.x.x
```

# EmotionHub 启动

## 前置条件（只需做一次）

```cmd
# 1. 初始化数据库（只需第一次）
Get-Content database/init.sql -Encoding UTF8 | docker exec -i emotionhub-mysql mysql -u root -p123456 --default-character-set=utf8mb4
```

---

## 每次启动

**第一步：启动 Docker 容器**

```cmd
cd D:\EmotionHub
docker compose up -d
```

 **第二步：启动后端** （新开一个终端）

```cmd
cd D:\EmotionHub\backend\emotion-hub-web
mvn spring-boot:run
```

 **第三步：启动前端** （再开一个终端）

```cmd
cd D:\EmotionHub\frontend
npm run dev
```

## 常见问题

### 1. 后端启动失败

**问题：端口8080被占用**

```
Port 8080 was already in use
```

解决方案：

- 修改 `application-dev.yml` 中的端口号

```yaml
server:
  port: 8081  # 改成其他端口
```

**问题：数据库连接失败**

```
Communications link failure
```

解决方案：

- 检查MySQL是否启动
- 检查用户名密码是否正确
- 检查数据库名称是否正确

**问题：Redis连接失败**

```
Unable to connect to Redis
```

解决方案：

- 检查Redis是否启动: `redis-cli ping`
- 如果Redis设置了密码，在配置文件中填写密码

### 2. 前端启动失败

**问题：依赖安装失败**

```
npm ERR! code ECONNREFUSED
```

解决方案：

```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```

**问题：端口3000被占用**

解决方案：

- 修改 `vite.config.ts`

```typescript
export default defineConfig({
  server: {
    port: 3001  // 改成其他端口
  }
})
```

### 3. 前后端连接失败

**问题：前端调用后端接口失败**

解决方案：

1. 确认后端已启动: http://localhost:8080/api/test/hello
2. 检查前端配置的API地址是否正确
3. 检查浏览器控制台是否有跨域错误
