# EmotionHub 项目启动指南

## 当前有效的启动方式

当前仓库支持两种启动方式，请二选一，不要混着启动：

1. 全量 Docker 运行
2. 本地开发运行（Docker 只起 MySQL 和 Redis）

`database/init.sql` 属于旧初始化方案，当前正常启动不需要手工执行。数据库结构和示例数据由 Flyway 自动初始化，MySQL 权限由容器启动脚本自动处理。

---

## 方式一：全量 Docker 运行

这是当前最省事、最接近可交付环境的启动方式。

### 启动

```powershell
cd D:\EmotionHub
.\start-docker.ps1
```

### 访问地址

| 地址                                 | 说明         |
| ------------------------------------ | ------------ |
| http://localhost:3000                | 前端页面     |
| http://localhost:8080/api/test/hello | 后端健康检查 |

说明：

- 前端容器通过 Nginx 反向代理 `/api` 到后端。
- 当前 `docker compose` 运行的是 `prod` 配置。
- `prod` 配置下 API 文档默认关闭，不要再按旧文档访问 `doc.html`。

### 停止

```powershell

docker compose down
```

如果你要彻底清空数据库和缓存卷，再重新生成一套全新测试数据：

```powershell
cd D:\EmotionHub
docker compose down -v
docker compose up -d --build
```

## 方式二：本地开发运行

这个方式适合改代码时热更新调试。

### 第一步：只启动依赖服务

```powershell
cd D:\EmotionHub
docker compose up -d mysql redis
```

### 第二步：启动后端

```powershell
cd D:\EmotionHub\backend\emotion-hub-web
mvn spring-boot:run
```

后端默认使用 `dev` 配置：

- 端口：`8080`
- MySQL：`localhost:3307`
- Redis：`localhost:6379`

对应配置见 [application-dev.yml](../backend/emotion-hub-web/src/main/resources/application-dev.yml)。

### 第三步：启动前端

首次启动先安装依赖：

```powershell
cd D:\EmotionHub\frontend
npm install
npm run dev
```

前端开发服务默认地址：

- http://localhost:3000

对应配置见 [vite.config.ts](../frontend/vite.config.ts)。

### 重要说明

- 不要在“全量 Docker 运行”已经启动 `frontend` 和 `backend` 的情况下，再本地执行 `mvn spring-boot:run` 或 `npm run dev`，否则会和 `8080`、`3000` 端口冲突。
- 如果你要切换到本地开发运行，建议先执行：

```powershell
cd D:\EmotionHub
docker compose stop frontend backend
```

### 本地开发可用地址

| 地址                                 | 说明                        |
| ------------------------------------ | --------------------------- |
| http://localhost:3000                | 前端开发页面                |
| http://localhost:8080/api/test/hello | 后端健康检查                |
| http://localhost:8080/doc.html       | API 文档（仅 dev 配置可用） |

## 验证启动是否成功

### 1. 验证后端

浏览器或命令行访问：

```powershell
curl http://localhost:8080/api/test/hello
```

预期返回包含：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": "Hello, EmotionHub!"
}
```

### 2. 验证登录

打开：

- http://localhost:3000/login

使用测试账号登录，例如：

- 用户名：`alice_chen`
- 密码：`password123`

### 3. 验证容器状态

如果使用 Docker 方式运行：

```powershell
cd D:\EmotionHub
docker compose ps
```

预期应看到：

- `emotionhub-mysql`
- `emotionhub-redis`
- `emotionhub-backend`
- `emotionhub-frontend`

## 常见问题

### 1. 登录时报 `network error`

优先检查下面两个地址是否正常：

- `http://localhost:8080/api/test/hello`
- `http://localhost:3000`

如果后端健康检查不通，先看后端是否正常启动。

### 2. 端口被占用

当前默认端口：

- 前端：`3000`
- 后端：`8080`
- MySQL：`3307`
- Redis：`6379`

如果端口冲突，先确认你是不是同时用了两套启动方式。

### 3. 数据库初始化失败

当前正常流程不需要手工执行 `database/init.sql`。

正确做法：

- Docker 方式：删除卷后重建
- 本地开发方式：确保 MySQL 容器正常，然后重启后端，让 Flyway 自动迁移

### 4. API 文档打不开

这是预期行为之一：

- `dev` 配置下可访问 `http://localhost:8080/doc.html`
- `prod` 配置下默认关闭 API 文档

---

## 推荐使用方式

- 只想把项目跑起来：用“全量 Docker 运行”
- 要改代码、调接口、看文档：用“本地开发运行”
